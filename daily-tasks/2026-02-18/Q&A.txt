3. Explain the concept of strides in NumPy. How do strides relate to memory layout?
=> in NumPy, strides tells you how many bytes to move in memory to reach the other element in the array. 
    example:
    a = np.array([[1, 2, 3],
              [4, 5, 6]], dtype=np.int32)

    print(a.strides)

    output: 
    (12,4) //12 bytes to move to the next row and 4 bytes to move to the next element in the same row. 

4. What is the difference between C-order (row-major) and Fortran-order (column-major) arrays?
=> c-order and f-order refers to how the elements are stored in the memory.
    c-order:
    in c-order, the elements of the rows are stored consequtively in the memory.
    f-order:
    in f-order, the elements of the columns are stored consequtively in the momory. 

    example(c-order):
    import numpy as np

    a = np.array([[1, 2, 3],
                [4, 5, 6]], order='C')

    memory layout:
    1,2,3,4,5,6

    example(f-order):
    a = np.array([[1, 2, 3],
              [4, 5, 6]], order='F')
    
    memory layout:
    1,4,2,5,3,5

8. What is the difference between np.stack(), np.hstack(), and np.vstack()?
=> np.hstack(): it stacks two arrays horizontally. 
    example:
    a = np.array([[1, 2, 3]])
    b = np.array([[4, 5, 6]])

    np.hstack([a, b])

    output:
    [[1 2 3 4 5 6]]

   np.vstack(): it stacks two arrays vertically. 
    example:
    a = np.array([[1, 2, 3]])
    b = np.array([[4, 5, 6]])

    np.vstack([a, b])

    output:
    array([[1, 2, 3],
       [4, 5, 6]])

   np.stack(): it joins arrays along a new dimension.

    example:
    for axis = 0

    a = np.array([1, 2, 3])
    b = np.array([4, 5, 6])

    np.stack([a, b], axis=0)

    output:
    array([[1, 2, 3],
       [4, 5, 6]])

    original_shape: (3,)
    new_shape: (2,3)

    for axis = 1

    np.stack([a, b], axis=1)

    output:
    array([[1, 4],
       [2, 5],
       [3, 6]])

    new_shape: (3,2)

PANDAS – THEORETICAL QUESTIONS

1. What is the difference between copy() and a view in Pandas? What is the SettingWithCopyWarning?
=> the difference between copy() and view in pandas is that a view of a pandas object points to the same memory as the object itself. so, changing the view can alter the original data itself. 
meanwhile, copy() will create a new memory and copy the object to the new memory. so alterning a copy of that object wont alter the original object. 

SettingWithCopyWarning:
    this is an error that occurs when we do index chaining in pandas.
    for example:
    df[df['age'] > 50]['salary'] = 5000

    here df[df['age'] > 50] will create a new object and ['salary'] will act upon that temporary object instead of the original df.
    Meaning, this warning occurs when pandas thinks we might be modifying a copy rather than the original object.

8. What is the difference between ffill() and bfill()?
=> both ffill() and bfill() is used to deal with timeseries data. 
    ffill():
    this means forward fill. this replaces the null/nan values with previous known values.

    example:
    arr = [100,200,nan,300]
    arr.bfill()
    output:
    [100,200,300,300]

    bfill():
    this means backward fill. this replaces the null/nan values with later known values. 

    example:
    arr = [100,200,nan,300]
    arr.ffill()
    output:
    [100,200,200,300]
